# AI多模态应用开发项目经历（简历版）

## 项目角色
后端开发 / AI应用开发（可写独立负责或核心参与）

## 项目简介（简历可直接使用）
面向儿童教育场景，设计并落地一套基于 FastAPI + MongoDB 的多模态 AI 应用后端，打通文本对话、图片理解、语音识别、数字人生成、成长档案生成与分享全链路。项目以 OpenAPI 为接口契约，支持前后端并行联调，并通过测试用例保障核心业务稳定性。

## 技术栈
- Python 3.10, FastAPI, Uvicorn
- MongoDB（PyMongo）
- JWT 鉴权（Bearer Token）
- OpenAI SDK（兼容 DashScope）
- 通义千问模型：qwen3-max、qwen3-vl-flash、qwen3-asr-flash
- 万相（WanX）图像/视频生成能力
- OSS 对象存储（导出与分享）
- Pytest（接口与业务回归测试）

## 个人职责与亮点（建议放简历）
1. 负责后端架构与接口设计，按模块拆分 routers/services/deps/schemas，形成可扩展的 AI 业务分层。
2. 基于 OpenAPI 规范落地约 40 个业务路由，覆盖鉴权、会话、上传、文档理解、语音、数字人、成长档案与分享等核心场景。
3. 实现多模态对话链路：
   - 文本对话：接入 qwen3-max，支持多轮上下文拼接。
   - 图片问答：接入 qwen3-vl-flash，支持图文联合理解。
   - 语音输入：接入 qwen3-asr-flash，完成语音转文本并回传前端继续对话。
4. 实现附件增强能力：对上传文件进行文本抽取，结合分块检索（Embedding/TF-IDF 兜底）构建轻量 RAG，提高回答相关性。
5. 设计会话与消息持久化方案（conversation/messages），支持会话创建、续聊、历史查询与软删除等能力。
6. 负责数字人能力接入：支持头像上传、卡通形象生成、图生视频任务创建与状态查询，满足互动展示需求。
7. 落地成长档案能力：支持 AI 生成成长故事、保存/更新/删除、导出 Word/PDF、分享链接生成与撤销。
8. 完成鉴权与安全控制：JWT 登录态校验、仅允许受控文件路径访问、上传文件服务与范围请求（Range）支持。
9. 建立测试与联调机制：编写/维护多模块测试用例（auth/chat/conversations/doc/avatar/profile_share 等），并配套调试脚本加速问题定位。

## 项目难点与解决（面试可展开）
1. 多模型调用稳定性：
   - 问题：外部模型接口存在超时、空响应、非预期返回。
   - 方案：统一超时控制、异常识别与用户友好降级文案，避免前端直接暴露底层错误。

2. 多模态输入复杂性：
   - 问题：语音格式不统一、附件类型多样、前端上送数据质量不稳定。
   - 方案：增加音频魔数检测、附件字段白名单校验、短音频拦截与类型兜底，提高链路鲁棒性。

3. RAG 可靠性：
   - 问题：Embedding 服务偶发不可用会影响文档问答。
   - 方案：设计 Embedding 检索 + TF-IDF 回退双路径，保证服务在弱依赖场景仍可用。

4. 业务闭环与可交付：
   - 问题：前端需要先联调，AI能力接入节奏不一致。
   - 方案：先保证响应结构稳定与契约一致（OpenAPI + 统一 ApiResponse），分阶段替换占位能力为真实模型调用。
